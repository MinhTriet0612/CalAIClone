### **MẪU ĐẶC TẢ USECASE TÍCH HỢP**

| Mục | Nội dung chi tiết (Ví dụ: UC-08 Phân tích ảnh món ăn) |
| :---- | :---- |
| **1\. Thông tin chung** |  |
| Mã Usecase | UC-08 |
| Tên Usecase | Phân tích ảnh món ăn bằng AI |
| Tác nhân (Actors) | Người dùng (User), Hệ thống AI (Google Gemini) |
| Mô tả ngắn | Người dùng tải ảnh bữa ăn lên để hệ thống nhận diện và bóc tách các chỉ số dinh dưỡng (Calo, Macro). |
| **2\. Điều kiện tiên quyết** | Người dùng đã đăng nhập và đang ở màn hình Dashboard. |
| **3\. Điều kiện đảm bảo** | Thông tin dinh dưỡng của món ăn được hiển thị chính xác lên màn hình xác nhận. |
| **4\. Luồng sự kiện chính (Basic Flow)** | 1\. Người dùng bấm nút "Thêm bữa ăn" và chọn tải ảnh lên. 2\. Hệ thống kiểm tra định dạng và dung lượng ảnh. 3\. Hệ thống gửi ảnh sang API Google Gemini. 4\. Gemini trả về dữ liệu JSON chứa Calo, Protein, Carbs, Fats. 5\. Hệ thống hiển thị kết quả lên màn hình Modal cho người dùng. |
| **5\. Luồng rẽ nhánh/ Ngoại lệ (Alternative Flows)** | **5a. Ảnh không hợp lệ (Tại bước 2):** Hệ thống báo lỗi dung lượng/định dạng và yêu cầu chọn lại. **5b. Ảnh không có đồ ăn (Tại bước 4):** AI trả về isFood \= false, hệ thống hiện thông báo "Không tìm thấy thức ăn" và hủy luồng. |
| **Yêu cầu chức năng (FR) để Dò vết** | Tất cả các FR dưới đây sẽ được dùng làm Cột 1 trong Ma trận Dò vết: 🔹 **FR\_08.1:** Hệ thống chỉ chấp nhận ảnh định dạng JPG/PNG/HEIC, tối đa 5MB. 🔹 **FR\_08.2:** Hệ thống phải giao tiếp được với Gemini API và bóc tách đúng 5 trường dữ liệu: isFood, calo, protein, carbs, fats. 🔹 **FR\_08.3:** Hệ thống phải chặn luồng và hiển thị cảnh báo UI nếu isFood \= false. |

### 

**Chiều xuôi (Forward):** Từ Use Case, ta suy ra các FR để đảm bảo không bỏ sót nhu cầu của người dùng.  
**Chiều ngược (Backward):** Mỗi FR được viết ra phải được "gắn thẻ" vào một Use Case cụ thể. Nếu một FR không thuộc Use Case nào, đó là yêu cầu dư thừa (Gold Plating).
